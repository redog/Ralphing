# Everything is a Ralph Loop — Geoffrey Huntley (Extended Philosophy)

Source: https://ghuntley.com/loop/
Published: January 17, 2026

## Key Thesis

Standard software practices build brick by brick — like Jenga. Ralph is a mindset: **programming the loop**, not the bricks.

Ralph isn't just about forwards (building autonomously) or reverse mode (clean rooming) — it's a generic pattern for ALL TASKS using context engineering.

> "I'm there as an engineer just as I was in the brick by brick era but instead am programming the loop, automating my job function and removing the need to hire humans."

## Core Pattern

Ralph = allocate the array (context window) with required specs + give it a goal + loop the goal.

Software is now clay on the pottery wheel — if something isn't right, throw it back on the wheel.

**Watch the loop.** That is where your personal development and learning will come from. When you see a failure domain — put on your engineering hat and resolve the problem so it never happens again.

## Human-in-the-Loop vs Full Auto

Even manually prompting (with CTRL+C to progress to the next task) is "still ralphing." Ralph is about getting the most out of how models work through context engineering.

## The Levels (Steve Yegge's Gas Town)

- Level 8 = Gas Town: spinning plates and orchestration
- Level 9 (Huntley's target) = autonomous loops that evolve products and optimize for revenue generation = "evolutionary software" = software factory

## The Divide

Two camps:
1. Rejecting AI entirely
2. Using Claude Code/Cursor to accelerate lego-brick building (still brick by brick)

Huntley's position: software development as a profession is dead (programmable cheaper than a burger flipper, AFK). Software *engineering* (programming the loop, understanding LLMs as a new programmable computer) is very much alive and more critical than ever.
